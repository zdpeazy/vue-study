### 7、内置的指令v- ###

	--v-bind : 动态绑定数据/属性 简写 :
	--v-on : 绑定事件监听 简写 @
	--v-text ： 更新数据，覆盖已有的结构
	--v-html ： 可以解析数据里面html结构
	--v-show ： 根据值得真假，切换元素的display属性
	--v-if ： 根据值得真假，切换元素会被销毁、重建
	--v-else-if ： 多条件判断，真则渲染
	--v-else ： 条件都不符合渲染
	--v-for ： 基于源数据多次渲染元素或者模板
	--v-model ： 在表单控件上创建双向数据绑定
	--v-pre ： 跳过元素和子元素的编译过程
	--v-once ： 只渲染一次，随后数据更新不在渲染
	--v-cloak ： 隐藏未编译的Mustache语法，css中设置[	           v-cloak]{display:none}

# 二、组件 #
### 1、基本概念 ###
	web中的组件就是页面中的一部分，好比电脑中的元件，它是具有独立的逻辑和功能或者界面，同时又能根据规定的接口规则互相融合，变成一个完整的应用
	页面是由一个个类似这样的部分组成，比如导航、列表、弹窗、下拉菜单等、页面只不过是这些组件的容器、组件自由组合成完整的页面、当不需要组件或者更换组件的时候，可以随时替换或者删除

### 2、前端组件化核心思想 ###

	将巨大复杂的东西拆成粒度合理的小东西

### 3、使用组件的好处
	--提高开发效率
	--方便重复使用
	--简化调试步骤
	--提升整个项目的可维护性
	--便于协同开发
### 4、组件的特性
	--高内聚
	组件的功能必须是完整的，如我要实现下拉菜单功能，那在下来菜单这个组件中就把下来菜单所需要的所有的功能都实现
	--低耦合
	代码独立不会和项目中其他的代码冲突！更易于团队合作

### 5、vue中的组件
	--自定义标签（得先注册）
	--组成
		样式结构
		行为逻辑
		数据

### 6、vue封装组件
	--注册组件
	--全局注册 Vue.component(组件名,选项对象)
	命名约定：

	在html中使用自定义标签组件

		}
	}

### 7、vue组件通信（传值）---组件的差异化
	父组件要给子组件传递数据，子组件需要将它内部发生的事情告知给父组件 
	--父组件-->子组件
		组件实例的作用域是鼓励的，不能直接在子组件直接用父组件的数据。
		可以在组件上使用自定义属性绑定数据。在组件需要显式的用props声明自定义属性名
	--子组件->父组件
		需要用到自定义事件，父组件用$on监听自定义事件,$emit触发父组件所关心的自定义事件
	例：<custom-select btn-value="查询"></custom-select>
		Vue.component()里面写：
		data: function(){
	        return {
	          selectShow: false,
	        };
	      },
	      props: ['btnValue'],
